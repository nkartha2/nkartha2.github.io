First Post: Being Mobile First : Length units

People in the design community have not been hinting at being mobile first so much as yelling it through the streets or sagely telling newcomers this phrase like it is a cautionary tale. As a newbie myself, I’ve heard it and now I need to practice it. But HOW? Here’s what I’ve picked up on (if you see or hear something incorrect or notice something missing in this picture, SAY SOMETHING…. Please:

1. Font-sizing: relative font-sizing

  1. Definitions:
  2. Relative length: amount of space that the content consumes is in relation to another length
  3. Absolute length: fixed size that does not change according to screen size; maintains size according to the prescribed unit value

Purpose: Using relative lengths will help scale between different output devices/mediums; for example, using relative units will help manage space between desktop, laptop, and mobile phone; used for “responsive design” Types: Relative : em, rem, ex, vh, vw, vmin, vmax Absolute: px, inches, millimeters, centimeters, points, picas

  1. Ex:
  2. Etymology:
    3. X-height
  3. Value:
    4. based on the size of the element’s font’s x size; does not resize with the window; heavily dependent on font-family
  4. Pros:
    5. possibly helpful in typesetting – fonts with the same ex size might be more legible ……………..help me out with more uses of this unit please
  1. Em:
  2. Etymology:
    3. Why is it called “em”? This used to be based on the width of the element’s typeface’s capital letter “M”; now it is based on vertical space a letter consumes – tangerine
  3. Value:
    4. Em length unit depends on the font size being used for the element. 1em is equal to the inherent font size. For example, most browsers default the font size to 16px, so 1em is equal to 16px until declared otherwise; therefore, it is not dependent on font-family but on font-size
  4. Pros:
    5. -used in responsive design -easy to manage space according to font size using ems – Kaplan
  5. Look out!
    6. *compounding effect: when elements are nested, one element within another, then the parent element font size declared in ems will affect the child element’s font size.
  1. Rem:
  2. Etymology:
    3. root em
  3. Value:
    4. rems are relative to the font-size of the html root element.
  4. Pros:
    5. This prevents the compounding effect because all following font sizes are relative to the root element rather than the parent element like we saw with ems.
  5. Look out!....kinda-
    6. not used in old browsers like IE 8.0, iOS Safari 3.2, Opera mini 5.0-7.0 (fallback would be to use px)
  1. Vh, vw, vmin, vmax
  2. Etymology:
    3. vw: viewport width vh: viewport height vmin: viewport minimum vmax: viewport maximum
  3. Value:
      4. 1vw is equal to 1% of the viewport width 1 vh is equal to 1% of the viewport height vmin is dependent on the shortest dimension whether it is width or height vmax is dependent on the longest dimension whether it is width or height
    1. Pros:
      2. no compounding effect; based on root element
    2. Lookout!:
      3. Limited browser support
